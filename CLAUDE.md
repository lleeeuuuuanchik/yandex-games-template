# CLAUDE.md

Этот файл — инструкция для Claude Code и Claude в Cursor при работе с этим репозиторием.

## Назначение репозитория

Шаблон для создания браузерных игр под платформу **Яндекс Игры** (https://yandex.ru/games/). Используется как стартовая точка: AI-ассистент получает описание игры от пользователя и реализует её на базе этого шаблона.

## Стек и ограничения

- **Vanilla JavaScript** — без фреймворков, без TypeScript, без сборщиков (Webpack/Vite/etc.)
- **HTML5 Canvas** — основной способ рендеринга игрового поля
- **CSS** — стилизация UI-экранов (меню, магазин, настройки)
- **GSAP + Draggable** (CDN) — анимации и drag-and-drop
- **Google Fonts** — шрифты
- **Нет npm/yarn/pnpm** — нет package.json, нет node_modules
- **Нет тестов/линтеров/CI** — простая статическая структура
- Все скрипты подключаются через `<script>` теги в `index.html` в порядке зависимостей
- Все модули — глобальные объекты (не ES modules)

## Запуск

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

Или открыть `index.html` напрямую в браузере.

## Yandex Games SDK

### Подключение
```html
<script src="https://yandex.ru/games/sdk/v2"></script>
```

### Обязательные требования
1. **`LoadingAPI.ready()`** — вызвать после полной загрузки игры (когда главное меню готово к взаимодействию). Без этого модерация не пройдёт.
2. **Реклама** — rewarded (за награду) и interstitial (между уровнями). Без SDK игра должна работать без рекламы (graceful degradation).
3. **SDK доступен только на домене Яндекса** — локально `YaGames` будет `undefined`, это нормально.

### Обёртка YandexSDK (src/yandex-sdk.js)
Готовая обёртка уже включена в шаблон. Использование:
```js
// Инициализация (один раз при старте)
YandexSDK.init(function(ok) { /* ok = true если SDK загружен */ });

// Сообщить о готовности
YandexSDK.notifyReady();

// Rewarded реклама
YandexSDK.showRewarded(onSuccess, onError);

// Interstitial реклама
YandexSDK.showInterstitial(onClose);
```

## Паттерны разработки

### Экраны
- Каждый экран — `<div class="screen" id="screen-xxx">`
- Переключение через добавление/удаление CSS-класса `.active`
- Анимации переходов через GSAP (fade + scale)

### Игровое состояние
- Объект `Game` хранит всё состояние (сетка, счёт, уровень, текущая фигура)
- `Progress` сохраняет/загружает из `localStorage`
- Ключ localStorage: `{game_id}_progress`

### Canvas рендеринг
- Основное игровое поле — `<canvas>`
- Объект `Render` отвечает за отрисовку
- `requestAnimationFrame` для анимаций/частиц

### Визуальный стиль
- Тёмная тема с glassmorphism (backdrop-filter: blur)
- Неоновые акценты (розовый, фиолетовый, голубой)
- Скруглённые элементы, градиенты, свечение (box-shadow)

## Выгрузка в Яндекс Игры

1. Собрать ZIP-архив с файлами в корне (НЕ вложенные в папку):
   ```
   zip -r game.zip index.html styles.css src/ assets/
   ```
2. Загрузить в консоль разработчика: https://games.yandex.ru/console/
3. ZIP проверяется ~15 минут, модерация — 3–5 рабочих дней

## Язык

- Весь UI и комментарии в коде — **на русском языке**
- Названия переменных и функций — на английском

## Важные правила при разработке

1. **Мобильная адаптация обязательна** — touch-события, адаптивная вёрстка, viewport meta
2. **Все цены/награды выносить в CONFIG** — не хардкодить в логике
3. **localStorage может быть недоступен** — оборачивать в try/catch
4. **Порядок скриптов в index.html важен** — зависимости загружаются первыми
5. **Нет ES modules** — только глобальные объекты и var/function
6. **GSAP подключается через CDN** — не копировать в проект

# CLAUDE.md

Этот файл — инструкция для Claude Code и Claude в Cursor при работе с этим репозиторием.

## Стиль кода

**Перед написанием любого кода прочитай `CodeStyle.md` в корне проекта и строго следуй всем соглашениям оттуда.** Это касается форматирования, именования, отступов, BEM, комментариев и структуры файлов.

## Назначение репозитория

Шаблон для создания браузерных игр под платформу **Яндекс Игры** (https://yandex.ru/games/). Используется как стартовая точка: AI-ассистент получает описание игры от пользователя и реализует её на базе этого шаблона.

## Стек и ограничения

- **Vanilla JavaScript** — без фреймворков, без TypeScript, без сборщиков (Webpack/Vite/etc.)
- **HTML5 Canvas** — основной способ рендеринга игрового поля
- **CSS** — стилизация UI-экранов (меню, магазин, настройки)
- **GSAP + Draggable** (CDN) — анимации и drag-and-drop
- **Google Fonts** — шрифты
- **Нет npm/yarn/pnpm** — нет package.json, нет node_modules
- **Нет тестов/линтеров/CI** — простая статическая структура
- Все скрипты подключаются через `<script>` теги в `index.html` в порядке зависимостей
- Все модули — глобальные объекты (не ES modules)

## Запуск

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

Или открыть `index.html` напрямую в браузере.

## Yandex Games SDK

### Подключение
```html
<script src="https://yandex.ru/games/sdk/v2"></script>
```

### Обязательные требования
1. **`LoadingAPI.ready()`** — вызвать после полной загрузки игры (когда главное меню готово к взаимодействию). Без этого модерация не пройдёт.
2. **Реклама** — rewarded (за награду) и interstitial (между уровнями). Без SDK игра должна работать без рекламы (graceful degradation).
3. **SDK доступен только на домене Яндекса** — локально `YaGames` будет `undefined`, это нормально.

### Обёртка YandexSDK (src/yandex-sdk.js)
Готовая обёртка уже включена в шаблон. Использование:
```js
// Инициализация (один раз при старте)
YandexSDK.init(function(ok) { /* ok = true если SDK загружен */ });

// Сообщить о готовности
YandexSDK.notifyReady();

// Rewarded реклама
YandexSDK.showRewarded(onSuccess, onError);

// Interstitial реклама
YandexSDK.showInterstitial(onClose);
```

## Паттерны разработки

### Экраны
- Каждый экран — `<div class="screen" id="screen-xxx">`
- Переключение через добавление/удаление CSS-класса `.active`
- Анимации переходов через GSAP (fade + scale)

### Игровое состояние
- Объект `Game` хранит всё состояние (сетка, счёт, уровень, текущая фигура)
- `Progress` сохраняет/загружает из `localStorage`
- Ключ localStorage: `{game_id}_progress`

### Canvas рендеринг
- Основное игровое поле — `<canvas>`
- Объект `Render` отвечает за отрисовку
- `requestAnimationFrame` для анимаций/частиц

## Выгрузка в Яндекс Игры

1. Собрать ZIP-архив с файлами в корне (НЕ вложенные в папку):
   ```
   zip -r game.zip index.html styles.css src/ assets/
   ```
2. Загрузить в консоль разработчика: https://games.yandex.ru/console/
3. ZIP проверяется ~15 минут, модерация — 3–5 рабочих дней

## Язык

## Важные правила при разработке

1. **Мобильная адаптация обязательна** — touch-события, адаптивная вёрстка, viewport meta
2. **Все цены/награды выносить в CONFIG** — не хардкодить в логике
3. **localStorage может быть недоступен** — оборачивать в try/catch
4. **Порядок скриптов в index.html важен** — зависимости загружаются первыми
5. **Нет ES modules** — только глобальные объекты и var/function
6. **GSAP подключается через CDN** — не копировать в проект

## Требования модерации Яндекс Игр (обязательные правила)

При разработке любой игры я ОБЯЗАН следовать этим правилам. Каждое решение проверяется на соответствие требованиям платформы. Я думаю как модератор Яндекс Игр и не предлагаю решения, которые приведут к отклонению.

### SDK и инициализация
- Вызывать `LoadingAPI.ready()` когда главное меню готово к взаимодействию
- Вызывать `GameplayAPI.start()` при начале геймплея и `GameplayAPI.stop()` при завершении/паузе
- Реклама вызывается ТОЛЬКО через SDK Яндекс Игр — никаких сторонних рекламных сетей
- Платежи ТОЛЬКО через SDK — никаких сторонних платёжных систем
- SDK недоступен локально — игра должна работать без SDK (graceful degradation)

### Звук и пауза
- **Звук ОБЯЗАН останавливаться** при сворачивании страницы/вкладки (и десктоп, и мобильные)
- При показе любой рекламы (interstitial/rewarded) — звук на паузе, геймплей на паузе
- Реализовать кнопку отключения звука
- Реализовать возможность паузы

### Мобильные устройства
- Полноэкранный режим во время игры
- Управление ТОЛЬКО жестами (touch) — никакой зависимости от клавиатуры
- Управление одной рукой — все основные действия доступны без прокрутки
- Долгое нажатие (лонгтап) НЕ должно вызывать выделение текста или контекстное меню
- Клавиатура показывается автоматически при нажатии на поле ввода
- Визуальные элементы НЕ деформируются при смене ориентации экрана
- Системный медиа-плеер НЕ должен появляться
- Убрать уведомление о WebGL при открытии
- Размеры кнопок достаточные — исключить случайные нажатия

### Десктоп
- Активное поле растягивается до края доступной области
- Соотношение сторон активного поля НЕ превышает 1:2
- Управление клавиатурой/мышью по умолчанию
- НЕ использовать горячие клавиши ОС (Ctrl+W, Ctrl+T, F5 и т.д.)
- Взаимодействие НЕ приводит к выделению текста или контекстному меню

### Адаптивность и вёрстка
- Масштаб НЕ выходит за границы экрана, элементы НЕ обрезаются
- Отсутствует браузерная прокрутка и swipe-to-refresh
- Элементы и тексты НЕ накладываются друг на друга
- CSS `user-select: none` на игровом поле
- CSS `overscroll-behavior: none` для предотвращения pull-to-refresh
- CSS `touch-action: manipulation` для предотвращения зума по двойному тапу
- `overflow: hidden` на body для предотвращения скролла

### Сохранение прогресса
- Для игр с прогрессом: сохранять СРАЗУ после действия или по кнопке
- Обновление страницы НЕ должно терять данные
- Для бесконечных игр: сохранять рекорд
- localStorage оборачивать в try/catch

### Локализация
- Минимум один язык, выбранный при публикации
- Автоопределение языка через SDK (`environment.i18n.lang`)
- Все тексты, влияющие на геймплей, переведены
- Если переключение языка вручную — оно должно быть доступно без знания текущего языка (иконки, а не текст)

### Контент — ЗАПРЕЩЕНО
- Мат и нецензурная лексика на ЛЮБОМ языке
- Порнография, эротика, сексуальный контент
- Оскорбления по полу, национальности, религии
- Эзотерика, гадания, предсказания о жизни/смерти/здоровье
- Реалистичное насилие, жестокость к детям/животным
- Политика, политические деятели, конфликты, войны
- Религия, религиозные деятели и атрибуты
- Азартные игры, лотереи, выигрыш реальных денег/призов
- Ссылки на внешние ресурсы (сайты, сторы, соцсети без согласования)
- Интерактивное использование ИИ в игре (предгенерированные ИИ-ассеты — ОК)
- YouTube-видео и любые видеоплееры с переходом на внешние ресурсы

### Реклама — правила
- Rewarded: ТОЛЬКО по желанию игрока (кнопка с описанием награды)
- Rewarded: награда — бонус, НЕ обязательное условие для продолжения игры
- Interstitial: ТОЛЬКО в логических паузах (между уровнями, после смерти)
- Реклама НЕ прерывает активный геймплей
- Допустимы стики-баннеры; кастомные RTB-баннеры ЗАПРЕЩЕНЫ

### Покупки (in-app)
- Только через SDK Яндекс Игр
- Подключить метод консумирования покупок
- Все цены указаны цифрами с обозначением портальной валюты
- Валюта определяется автоматически через SDK
- Изображение и описание покупки соответствуют реальному содержимому

### Ссылки
- ЗАПРЕЩЕНЫ ссылки на внешние ресурсы (сайт, сторы, другие платформы)
- ЗАПРЕЩЕНЫ ссылки в логотипах и графике
- ЗАПРЕЩЕНО автоматическое перенаправление на внешние сайты
- Ссылки на другие игры — ТОЛЬКО через SDK, ТОЛЬКО на каталог Яндекс Игр
- Исключение: ссылки на сообщества с "Яндекс Игры" в названии, без внешних ссылок

### Качество и стабильность
- Нет технических сообщений об ошибках, вылетов, зависаний
- Нет ошибок при повороте экрана, лонгтапе, сворачивании, открытии рекламы
- Минимум ошибок в консоли браузера
- Игра имеет завершённый вид — не альфа/бета
- Основной контент — минимум 10 минут геймплея
- Казуальная игра: 10–20 уровней; викторина: 100+ уникальных вопросов

### Файлы и архив
- Общий размер в разархивированном виде — НЕ более 100 МБ
- `index.html` в корне архива
- Имена файлов/папок — без пробелов и кириллицы
- НЕ использовать абсолютные URL на серверы S3 Яндекса

### Авторизация
- НЕ требовать регистрацию на сторонних сервисах
- Допустима авторизация через Яндекс ID — ТОЛЬКО по кнопке (не автоматически)
- Обязателен гостевой вход с сохранением прогресса

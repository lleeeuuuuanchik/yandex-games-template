# Как я пишу код

Личные соглашения по форматированию, именованию и структуре — без привязки к конкретному фреймворку или репозиторию.

---

## JavaScript

- Переменные и функции — **camelCase**; константы — **SCREAM_CASE** или camelCase, единообразно в файле.
- Фигурные скобки в стиле **Allman**: открывающая `{` на новой строке, на одном уровне с `if` / `function` / блоком объекта.
- Для коротких веток **`if` / `for` / `while`** тело без фигурных скобок — нормально, если читается в одну-две строки.
- В конце строк **не оставляю** лишние пробелы.

Пример:

```javascript
const calculateTotal = (items) => {
	let total = 0;
	for (let i = 0; i < items.length; i++)
		total += items[i].price;

	return total;
};

if (user.isAuthenticated) {
	if (user.hasPermission('admin'))
		showAdminPanel();
}
else
	redirectToLogin();
```
## SCSS / CSS

- Скобки — снова **Allman**: открывающая `{` с новой строки после селектора.
- Одно простое свойство в блоке можно **сжать в одну строку** вместе с селектором, если так короче и яснее.
- **Вложенность не глубже трёх уровней**; специфичность держу за счёт BEM, а не каскада.
- Цвета, шрифты, веса, брейкпоинты — через **переменные** из общего файла переменных; если нужного нет — **добавляю переменную**, а не хардкожу значение в компоненте.
- Повторяющиеся вещи (переходы, медиазапросы) — через **миксины**, а не копипаст.

### BEM

- Блок: `.user-card`.
- Элементы: полное имя по смыслу, например `.user-card__header-avatar`, а не урезанное `.user-card__avatar`, если аватар относится к шапке карточки.
- Модификатор: `.user-card--featured`.
- Общие утилитарные классы, которые трогают весь проект — с префиксом **`g-`** (например `.g-icon`, `.g-container`).

### Символ `&` в SCSS

- **Использую** для модификаторов (`&--active`), псевдоклассов (`&:hover`), псевдоэлементов (`&::before`), состояний (`&.is-open`).
- **Не связываю** элементы через `&__something` — пишу отдельным селектором `.block__something`.

### Утилитарные классы (типа Tailwind)

- Могу смешивать с BEM и SCSS там, где это уже принято в файле; не превращаю разметку в сплошную стену утилит, если рядом ведётся нормальный BEM.

## Имена файлов и папок

- **Папки** — kebab-case: `user-profile`, `news-list`.
- **Обычные файлы** (не компоненты) — kebab-case: `user-service.js`, `api-utils.js`.
- **Компоненты Vue** — PascalCase: `UserCard.vue`, `NewsList.vue`.

---

## Комментарии и документация

- Комментирую **почему** и неочевидные решения, а не то, что и так видно из кода.
- Утилиты и функции с нетривиальным контрактом оформляю **JSDoc** (`@param`, `@returns`, при необходимости `@throws`).
- Сложную логику в паре мест снабжаю короткими пояснениями, а не построчным пересказом.

---

## Перед коммитом сам с себя прошу

- Табы и отступы ровные, **без** хвостовых пробелов.
- Новая типографика и цвета — **через переменные**, не «магические числа» в компоненте.
